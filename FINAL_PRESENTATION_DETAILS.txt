============================================================
1) PROJECT OVERVIEW (WHAT IT IS)
============================================================
This project is a complete pothole detection and analysis system. It uses
computer vision and deep learning to detect potholes in video frames and
estimate their depth, then classifies severity. The system has two main
parts:
- Backend (Python): detection, depth estimation, tracking, severity
- Frontend (Flutter): mobile UI that receives alerts (WebSocket)

Key idea: two neural networks are used in sequence:
1) YOLOv8 for object detection (find potholes)
2) MiDaS for depth estimation (estimate pothole depth)

Everything else is classical algorithms (tracking, geometry, thresholds).

============================================================
2) PROBLEM STATEMENT (WHY IT MATTERS)
============================================================
- Potholes cause accidents, vehicle damage, and maintenance costs.
- Manual inspection is slow and expensive.
- A real-time detection system can help authorities and drivers identify
  dangerous road segments faster and prioritize repairs.

============================================================
3) PROJECT GOALS (OBJECTIVES)
============================================================
- Detect potholes in video streams reliably and in real time.
- Estimate physical severity (depth-based) for prioritization.
- Provide an easy-to-use front end for live monitoring and alerts.
- Offer configurable presets for speed vs accuracy.
- Provide a modular, professional codebase for future expansion.

============================================================
4) HIGH-LEVEL ARCHITECTURE
============================================================
Dual Neural Network Pipeline:

Video Stream
   -> Frame Processing
   -> YOLOv8 Detection (ANN #1)
   -> Extract pothole regions (ROIs)
   -> MiDaS Depth Estimation (ANN #2)
   -> Severity Classification (rules + thresholds)
   -> IoU-based Tracking (temporal smoothing)
   -> Visualization + Output
   -> Optional WebSocket Stream to Frontend

============================================================
7) NEURAL NETWORK #1 - YOLOv8 (OBJECT DETECTION)
============================================================
Purpose: locate potholes in each frame.
Input: full frame (default 640x640, configurable)
Output: bounding boxes + confidence scores

Model notes (from docs):
- YOLOv8n or YOLOv8s (nano/small) variants
- Single class: pothole
- Typical metrics (test set):
  Precision ~85-92%, Recall ~80-88%, mAP@0.5 ~0.85

Key parameters:
- confidence_threshold (default balanced: 0.35)
- iou_threshold (default 0.45)
- max_detections per frame (default 10)

============================================================
8) NEURAL NETWORK #2 - MiDaS (DEPTH ESTIMATION)
============================================================
Purpose: estimate depth of pothole regions.
Input: cropped ROI from detected pothole bounding box
Output: relative depth map

MiDaS is used to provide depth cues from monocular RGB images. The depth
is combined with geometric estimation to generate a final severity score.

============================================================
9) DEPTH ESTIMATION + SEVERITY CLASSIFICATION
============================================================
Hybrid depth approach:
- Neural depth (MiDaS)
- Geometric depth (camera calibration, pinhole model)
- Combined using weighted fusion (example weights: 0.6 neural, 0.4 geo)

Severity Levels (depth-based thresholds):
- CRITICAL  : > 15 cm
- DANGEROUS: 10-15 cm
- MODERATE : 6-10 cm
- MINOR    : 3-6 cm
- SURFACE  : < 3 cm

============================================================
10) TRACKING AND TEMPORAL SMOOTHING
============================================================
- Uses IoU (Intersection over Union) to match detections between frames.
- Smooths bounding box positions across a buffer of frames.
- Reduces jitter and false positives in real-time output.

============================================================
11) PERFORMANCE PRESETS
============================================================
Presets in config:
- accuracy: lower confidence, no frame skip, best quality
- balanced: default, good balance
- speed: higher confidence, frame skip, smaller inference size
- cpu: reduced inference size, FP16 disabled, frame skipping

============================================================
12) FRONTEND (FLUTTER APP)
============================================================
Purpose: display live pothole alerts and statuses.
Connection: WebSocket over WiFi to backend server.

Key functionality:
- Connect to backend using IP address
- Live alerts with severity labels
- Color-coded severity display
- Real-time data streaming from backend

============================================================
13) WEBSOCKET COMMUNICATION
============================================================
Why WebSocket (per docs):
- More reliable than Bluetooth
- Faster and easier to connect
- Supports multiple devices

Default port: 8765
Flow:
- Start backend with --websocket
- Phone connects to ws://<PC_IP>:8765
